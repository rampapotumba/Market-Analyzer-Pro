# Market Analyzer Pro — Техническое задание v2.0

> Дата: 16 марта 2026 | Статус: В разработке

## 1. Общее описание

Market Analyzer Pro — аналитическая платформа для комплексного анализа финансовых рынков и генерации торговых сигналов. Система объединяет технический, фундаментальный, макроэкономический и геополитический анализ для формирования взвешенных торговых рекомендаций.

### 1.1. Целевые рынки

| Рынок | Инструменты | Фаза |
|-------|-------------|------|
| Форекс | EUR/USD, GBP/USD, USD/JPY, USD/CHF, AUD/USD, EUR/GBP, NZD/USD + кросс-пары | Фаза 1 |
| Акции US | S&P 500 компоненты, NASDAQ-100, индексы SPY/QQQ/DIA | Фаза 1 |
| Акции EU | DAX, FTSE 100, основные blue chips | Фаза 2 |
| Криптовалюты | BTC, ETH, SOL, XRP, ADA + топ-20 по капитализации | Фаза 1 |
| Commodities | Золото (XAU), нефть (WTI, Brent), серебро | Фаза 3 |

### 1.2. Горизонты торговли

| Горизонт | Таймфрейм | Время удержания | Веса анализа (TA / FA / Sent / Geo) |
|----------|-----------|-----------------|--------------------------------------|
| Скальпинг | M1, M5, M15 | Минуты — часы | 0.90 / 0.00 / 0.05 / 0.05 |
| Дейтрейдинг | M15, H1 | Часы — 1 день | 0.70 / 0.10 / 0.15 / 0.05 |
| Свинг | H4, D1 | 2-14 дней | 0.45 / 0.25 / 0.20 / 0.10 |
| Позиционный | D1, W1 | Недели — месяцы | 0.20 / 0.40 / 0.25 / 0.15 |

### 1.3. Выходные данные (торговый сигнал)

Каждый сигнал содержит:
- **direction**: LONG / SHORT / HOLD
- **entry_price**: точка входа
- **stop_loss**: уровень SL (ATR-based)
- **take_profit_1**, **take_profit_2**: целевые уровни
- **risk_reward**: соотношение R:R
- **position_size_pct**: % от депозита
- **confidence**: 0-100%
- **horizon**: ожидаемое время удержания
- **composite_score**: [-100, +100]
- **reasoning**: JSON с обоснованием (ключевые факторы)
- **indicators_snapshot**: JSON со значениями всех индикаторов

### 3.2. Весовая система TA

```
TA_Score = Σ (indicator_signal × weight × strength)
indicator_signal ∈ [-1, +1]
weight — настраиваемый (сумма = 1.0)
strength ∈ [0, 1]
```

## 6. Движок сигналов (Signal Engine)

### 6.1. Композитная оценка

```
Composite = W_ta × TA_Score + W_fa × FA_Score + W_sent × Sentiment + W_geo × Geo
Диапазон: [-100, +100]
```

### 6.2. Пороговые значения

| Диапазон | Сигнал | Действие |
|----------|--------|----------|
| +60...+100 | STRONG BUY | Long полный размер |
| +30...+59 | BUY | Long уменьшенный размер |
| -29...+29 | HOLD | Не открывать |
| -59...-30 | SELL | Short уменьшенный размер |
| -100...-60 | STRONG SELL | Short полный размер |

### 6.3. Risk Management

```
Stop Loss    = Entry ± ATR(14) × 1.5
Take Profit 1 = Entry ± ATR(14) × 2.0  (R:R = 1:1.33)
Take Profit 2 = Entry ± ATR(14) × 3.5  (R:R = 1:2.33)
Position Size = Risk% × Account / SL_Distance
Max Risk per trade: 1-2% депозита
Correlation Filter: Pearson > 0.7 → не дублировать позиции
```

### 6.4. Мульти-таймфрейм фильтрация

Сигнал на рабочем TF подтверждается старшим:
- Совпадение 2 старших TF: ×1.2 (усиление)
- Совпадение 1 старшего TF: ×1.0 (без изменений)
- Противоречие 1 старшего TF: ×0.7 (ослабление)
- Противоречие 2 старших TF: ×0.4 (сильное ослабление)

## 7. База данных и трекинг сигналов

### 7.1. Жизненный цикл сигнала

```
CREATED → ACTIVE → TRACKING → COMPLETED → ANALYZED
```

- **CREATED**: сигнал сгенерирован и записан в БД
- **ACTIVE**: цена достигла точки входа (или ≤1% от неё)
- **TRACKING**: позиция "открыта" (виртуально), мониторим
- **COMPLETED**: достигнут SL/TP или истёк срок
- **ANALYZED**: записан результат, обновлена статистика

## 8. Модуль анализа точности (Accuracy Tracker)

### 8.1. Требования

| ID | Описание | Приоритет |
|----|----------|-----------|
| AT-01 | Автоматическое отслеживание цены после сигнала | MUST |
| AT-02 | Фиксация Entry, SL, TP1, TP2 момента достижения | MUST |
| AT-03 | Расчёт P&L для каждого сигнала | MUST |
| AT-04 | Win Rate, средний профит/убыток | MUST |
| AT-05 | Разбивка по рынку, инструменту, TF, типу | MUST |
| AT-06 | Profit Factor | MUST |
| AT-07 | Sharpe Ratio | SHOULD |
| AT-08 | Max Drawdown | SHOULD |
| AT-09 | Анализ вклада модулей (TA/FA/Sent) в точность | SHOULD |
| AT-10 | Авто-калибровка весов | Фаза 3 |

### 8.2. Целевые метрики

| Метрика | Формула | Цель |
|---------|---------|------|
| Win Rate | Wins / Total × 100% | ≥ 55% |
| Profit Factor | Gross Profit / Gross Loss | ≥ 1.5 |
| Avg R:R Actual | Avg Win / Avg Loss | ≥ 1.3 |
| Sharpe Ratio | (Mean Return - Rf) / StdDev | ≥ 1.0 |
| Max Drawdown | Max peak-to-trough | ≤ 15% |
| Expectancy | (W% × AvgW) - (L% × AvgL) | > 0 |
